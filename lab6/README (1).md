## Описание проекта

**Трекер привычек** — это веб-приложение на PHP, позволяющее пользователю добавлять, сохранять и просматривать личные привычки. Приложение реализует полный цикл работы с данными: форма ввода → серверная валидация → сохранение в JSON-файл → отображение списка.

---

## Структура проекта

```
/
├── index.html      — Форма добавления новой привычки
├── save.php        — Обработка формы, валидация, сохранение данных
├── list.php        — Список всех сохранённых привычек с сортировкой
└── data.json       — Хранилище данных (создаётся автоматически)
```

## Функциональность

### `index.html` — Форма добавления привычки

Страница содержит HTML-форму для ввода новой привычки со следующими полями:

| Поле | Тип | Описание |
|------|-----|----------|
| Название | `text` | Название привычки (обязательное) |
| Описание | `textarea` | Подробное описание (необязательное) |
| Категория | `select` | Одна из 8 категорий |
| Сложность | `select` | Лёгкая / Средняя / Сложная |
| Дата начала | `date` | Дата старта привычки |
| Цель (дней) | `number` | Сколько дней выполнять |
| Дни выполнения | `checkbox` | Пн–Вс, минимум один день |

Форма отправляется методом `POST` на `save.php`.

---

### `save.php` — Сохранение и валидация

Файл принимает данные из формы, выполняет серверную валидацию и сохраняет запись в `data.json`.

#### Правила валидации

- **Название:** обязательное, от 3 до 100 символов
- **Описание:** необязательное, максимум 1000 символов
- **Категория:** должна быть одним из допустимых значений (`health`, `sport`, `education`, `productivity`, `mindfulness`, `social`, `finance`, `other`)
- **Сложность:** `easy`, `medium` или `hard`
- **Дата начала:** формат `YYYY-MM-DD`, обязательная
- **Цель (дней):** целое число от 1 до 365
- **Дни выполнения:** хотя бы один день из `mon`, `tue`, `wed`, `thu`, `fri`, `sat`, `sun`

#### Логика сохранения

1. Если валидация прошла успешно — формируется запись с уникальным `id` (`uniqid`) и временной меткой `created_at`
2. Существующий `data.json` считывается и декодируется
3. Новая запись добавляется в массив и файл перезаписывается с флагами `JSON_PRETTY_PRINT | JSON_UNESCAPED_UNICODE`
4. При ошибке записи выводится соответствующее сообщение

#### Вывод результата

- **Успех:** отображается таблица с данными сохранённой привычки
- **Ошибки:** выводится список ошибок валидации с возможностью вернуться к форме

### `list.php` — Список привычек

Страница загружает все записи из `data.json` и отображает их в виде таблицы.

#### Возможности

- **Сортировка** по любому полю: название, категория, сложность, дата начала, цель (дней), дата добавления
- Направление сортировки: по возрастанию (`asc`) / по убыванию (`desc`), переключается кликом на заголовок
- **Статистика** вверху страницы:
  - Общее количество привычек
  - Количество уникальных категорий
  - Средняя цель в днях
- Цветовая подсветка сложности (зелёный / жёлтый / красный)
- Обрезка длинных описаний до 80 символов с многоточием (без использования `mbstring`)

#### Параметры URL

| Параметр | Значение по умолчанию | Описание |
|----------|-----------------------|----------|
| `sort` | `created_at` | Поле сортировки |
| `order` | `desc` | Направление сортировки |

Пример: `list.php?sort=difficulty&order=asc`

## Безопасность

- Все выводимые данные экранируются через `htmlspecialchars()`
- Значения категории, сложности и дней проверяются по белому списку (`in_array(..., true)`)
- Параметр `sort` в `list.php` проверяется по допустимым полям
- Данные не попадают в базу данных — используется только файловое хранилище

1. Скопировать файлы в директорию веб-сервера (например, `htdocs` или `www`)

3. Открыть в браузере: `http://localhost/index.html`
4. Убедиться, что у PHP есть права на запись в директорию (для создания `data.json`)
