## 📌 Описание проекта

Веб-приложение для управления финансовыми транзакциями, реализованное на чистом PHP без фреймворков. Приложение позволяет добавлять, удалять, искать и сортировать транзакции, а также отображает галерею изображений из локальной директории.

---

## ⚙️ Функциональность

### Основные функции

| Функция | Описание |
|---|---|
| `calculateTotalAmount()` | Подсчёт общей суммы всех транзакций через `array_reduce` |
| `daysSinceTransaction()` | Вычисление количества дней с даты транзакции до текущей даты через `DateTime::diff` |
| `addTransaction()` | Добавление новой транзакции в глобальный массив |
| `deleteTransaction()` | Удаление транзакции по ID через `array_filter` |
| `findTransactionByDescription()` | Поиск транзакций по описанию (регистронезависимый, `stripos`) |
| `sortByDateDesc()` | Сортировка по дате (убывание) через `usort` + `strtotime` |
| `sortByAmountDesc()` | Сортировка по сумме (убывание) через `usort` |

### Обработка GET-запросов

| Параметр | Действие |
|---|---|
| `?add=1&id=...&date=...&amount=...&description=...&merchant=...` | Добавить транзакцию |
| `?delete={id}` | Удалить транзакцию по ID |
| `?search={текст}` | Фильтрация по описанию |
| `?sort=date` | Сортировка по дате |
| `?sort=amount` | Сортировка по сумме |

## 🗂️ Структура данных

Каждая транзакция — ассоциативный массив со следующими полями:

```php
[
    "id"          => int,     // Уникальный идентификатор
    "date"        => string,  // Дата в формате YYYY-MM-DD
    "amount"      => float,   // Сумма
    "description" => string,  // Описание
    "merchant"    => string   // Получатель/магазин
]
```

### Тестовые данные

```php
["id"=>1, "date"=>"2024-01-10", "amount"=>1200.50, "description"=>"Парфюм",   "merchant"=>"Ovico"],

["id"=>3, "date"=>"2023-12-01", "amount"=>2500.00, "description"=>"Наушники", "merchant"=>"BOMBA"],
```

## 🔍 Особенности реализации

- Используется **`declare(strict_types=1)`** — строгая типизация
- Все функции используют **стрелочные функции** (`fn()`) — синтаксис PHP 7.4+
- Сортировка и фильтрация работают **напрямую на глобальном массиве** через `global $transactions`
- По умолчанию (без параметра `sort`) список сортируется **по дате убывания**
- Удаление транзакции сопровождается **`confirm()`-диалогом** на стороне клиента

## ⚠️ Недостатки и возможные улучшения

| Проблема | Рекомендация |
|---|---|
| Данные хранятся в массиве (сбрасываются при каждом запросе) | Подключить базу данных (MySQL / SQLite) |
| Нет валидации входных данных на сервере | Добавить проверку и экранирование (`htmlspecialchars`, `filter_input`) |
| Использование `global` в функциях | Перейти к передаче массива по ссылке или использовать класс |
| XSS-уязвимость при выводе данных через `<?= ?>` | Экранировать вывод через `htmlspecialchars()` |
| GET-запросы для мутирующих операций (add, delete) | Использовать POST-запросы для изменения данных |
